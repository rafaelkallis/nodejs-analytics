NodeJS Analytics

by Rafael Kallis

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient. The Node.js package ecosystem, npm, is the largest ecosystem of open source libraries in the world.

Below you can find a network analysis of the Node.js github repository. The analysis is focused exclusively on github users which have contributed to the Node.js repository, or have interacted with a contributor. Please note that data from not registered users is not included in the analysis.

The analysis is based on data extracted by me from the public GitHub API using my homemade github crawler. All content is open source and may be used freely for any purpose.

Abstract overview over the collected data. In the core we have the contributors. Edges are directed and represent a "follows" relationship. In the outer ring, we have users which have a relationship to contributors, but didn't contribute to Node.js

Betweenness

Betweenness centrality quantifies the number of times a node acts as a bridge along the shortest path between two other nodes.

\Large C_B(v)=\huge\sum\limits_{\large s \ne v \ne t \in V} \Large \frac{\sigma_{st}(v)}{\sigma_{st}}

Where \footnotesize \sigma_{st} is the total number of shortest paths from node \footnotesize s to node \footnotesize t and \footnotesize \sigma_{st}(v) is the number of those paths that pass through \footnotesize v. All following values are normalised.

  • Median betweenness: \scriptsize 7.818556 \times 10^{-7}

Indegree Closeness

The indegree closeness centrality of a node is a measure of centrality in a network, calculated as the sum of the length of the shortest paths between the node and all other nodes in the directed graph. Thus the more central a node is, the closer it is to all other nodes.

\Large C(x)=\frac{N - 1}{\sum_yd(y,\space x)}

Where \footnotesize d(y,x) is the distance between vertices \footnotesize x and \footnotesize y.
